# Code of Conduct

## Standard

Project participation should stay technical, evidence-led, and respectful.
Disagreements about claims, proofs, or implementation choices should be resolved
against committed artifacts and reproducible commands.

## Unacceptable Conduct

- Harassment, threats, or personal attacks.
- Publishing private security reports before coordinated disclosure.
- Deliberately misrepresenting proof status, license scope, or blocker state.

## Enforcement

Report conduct concerns to `architects@zer0pa.ai`. The project owner may close,
hide, or reject issues, pull requests, or comments that violate this policy.
