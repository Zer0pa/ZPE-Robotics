# Security Policy

This repo is private staged. Do not open a public issue for a security problem.

## Report A Vulnerability

Email `architects@zer0pa.ai` with:

- affected file or component
- reproduction details or proof of concept
- impact and severity assessment
- any suggested remediation

## In Scope

- secrets or credentials committed into the repo
- dependency issues in declared package dependencies
- workflow or release-process weaknesses that could expose private material

## Out Of Scope

- correctness failures in the robotics proofs
- disagreements with the technical verdict
- historical path residue in preserved historical bundles

Use the normal issue path for engineering defects and evidence disputes.
